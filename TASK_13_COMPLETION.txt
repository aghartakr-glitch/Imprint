VALIDATION COVERAGE:
====================

Each validator checks:
- Required field presence
- Type validation (string, number, boolean)
- Format validation (regex patterns for IDs, ISO8601 timestamps)
- Enum validation (strict values from allowed_values)
- Range validation (e.g., satisfaction_score 1-5)
- Percentage format validation (XX% or "unknown")

Example validations implemented:
- raw_id matches format: raw_YYYYMMDD_HHMMSS
- experiment_id matches format: exp_YYYYMMDD_HHMMSS
- feedback_unit_id matches format: exp_YYYYMMDD_HHMMSS_fu##
- patch_id matches format: exp_YYYYMMDD_HHMMSS_p##_##
- percentage values must be XX% (1-3 digits + %)
- satisfaction_score must be 1-5
- enum fields strictly checked (no typos)

SCHEMA RULES DOCUMENTED:
========================

- experiment_id_uniqueness: Never duplicate experiment IDs
- append_only: Sheets 01, 03, 08, 10, 11, 14 are append-only
- upsert_allowed: Sheets 02, 04, 05, 06, 07, 09, 12, 13 allow upsert
- percentage_format: All percentages in XX% format
- id_format: All IDs follow their specified format exactly
- no_data_corruption: Existing rows never overwritten
- referential_integrity: Foreign keys reference existing records
- timestamp_consistency: Consistent ISO8601 timestamps
- numeric_ranges: Satisfaction 1-5, match scores 0-100%
- enum_strict: No variations or typos in enum values

DATA FLOW DOCUMENTED:
=====================

Step-by-step flow from user feedback to rules:
1. User submits feedback -> 01-Raw Experiment Log + 02-Experiment Summary
2. System parses feedback -> 03-Feedback Unit Log
3. System interprets units -> 04-Variable Patch Log
4. System applies patches -> 05-Before After Values
5. System checks for side effects -> 06-Lock Check
6. System scores result -> 07-Score Breakdown
7. System logs failures -> 08-Failure Analysis
8. System applies/updates rules -> 09-Rule Memory, 10-Rule Application Log
9. Researcher codes insights -> 11-Research Coding
10. System tracks variables -> 12-Variable Dictionary
11. System maintains input schema -> 13-Auto Input Schema
